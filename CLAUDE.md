# Трекер тренировок

Рабочее пространство для планирования и отслеживания силовых тренировок: **HTML-приложение на телефон (Apps Script + Google Sheets) + Claude Code как тренер-аналитик**.

## Конфиденциальность
- Персональные данные (имя, возраст, вес, травмы, ID таблицы, URL приложения, секрет, название зала) — ТОЛЬКО в `PRIVATE.md` (в `.gitignore`).
- Перед коммитом: `git diff --cached` не должен содержать имён, медданных, ID внешних ресурсов.
- `CLAUDE.md` — публичный: только структура, форматы, правила.

## Архитектура

- **Приложение** — `app/` (одностраничный HTML, хостится самим Apps Script). Контракт данных — **`app/DATA-CONTRACT.md`** (источник правды по структуре).
- **Планы = вкладки `План: <название>`** в таблице (напр. `План: fullbody`, `План: Восстановление`). Приложение их находит (`listPlans`), пользователь выбирает в настройках ⚙.
- **Факт** пишется в вкладку **«Сессии»** (по подходам). Отсюда Claude берёт данные для прогрессии.
- **Активная** — legacy (исходная история), новым флоу не используется.
- Деплой/обновление приложения — `app/README.md` (через `clasp`).

### Формат вкладки «План: …» (широкий, по подходам)
`A=День(номер) | B=Упражнение (ключ) | C=Разминка(да/пусто) | D+=Неделя N`. Строка = один подход (упражнения дублируются по подходам, вкл. разминочные). Ячейка = `повт×вес` (`8×120`, `10`, `8–12×40`, `12×90lbs`, `10×тонк`; `—`/пусто = подхода нет). Дни — через 2 пустые строки. **Заметки-контекст** (отдых/RIR/техника) — комментарии (cell notes) на колонке B; приложение показывает их 💡.

### Формат «Сессии»
`ИД сессии | План | Неделя | Дата | День | Упражнение | № подхода | Разминка | Повторы | Вес | RPE | Заметка | Статус | Самочувствие | Время записи`. `Самочувствие` — RPE всей тренировки (1–10).

## Скилы проекта (`.claude/skills/`, запускать claude из папки проекта)
- **`new-plan`** — создать новую программу (вопросы → анализ прошлого → вкладка `План: <name>` с заметками).
- **`plan-next-week`** — прогрессия: факт из «Сессии» → новая неделя новой колонкой в плане (по-недельно/по-дневно).
- **`setup`** — первичная настройка (профиль → PRIVATE.md → первый план через `new-plan`).

## Правила для ассистента
- Перед изменением плана — уточнять: объём, частота, симптомы, допустимые/запрещённые движения.
- Не давать медицинских назначений; при тревожных симптомах — к врачу.
- Учитывать ограничения из `PRIVATE.md`.
- Язык общения — русский; комментарии в коде и git-сообщения — английский.

## Принципы тренировок (40+)
- **Full-body** 2–3 раза/нед эффективнее сплита. Силовые + кардио (~150 мин/нед зоны 2).
- Базовые многосуставные в приоритете; баланс тяг/жимов ~2:1 (здоровье плеч).
- Восстановление: сон, белок 1.6–2.2 г/кг.

## Параметры и структура тренировки
- 45 мин: full-body 3×/нед + кардио 2×/нед. Разминка плеч резинкой обязательна; кор в начале.
- Отдых: база 90–120 сек, hip thrust 120–180, изоляция 60–90.
- Разминочные подходы: тяжёлые 40/65/85%; средние 50/75%; лёгкие/высокоповторные 1×50%.

## Правила прогрессии
- **Двойная прогрессия:** достиг верха диапазона повторов во ВСЕХ рабочих подходах с запасом (RPE ≤ 8 / RIR 1–2) → +вес (жим ногами +5–10, верх тела +2.5–5, hip thrust +5, изоляция меньше), повторы → низ диапазона. Иначе тот же вес, +1–2 повтора.
- **Самочувствие/RPE недели:** 7–8 или боль → консервативно; 9–10 → deload.
- **Deload каждые 4–6 недель:** −30–40% объёма, −20–30% веса, доборы убрать.
- **Ограничения:** жимы/подтягивания RIR 2–3 (не в отказ); поясница; плечо (без горизонтальных жимов, нейтральный хват).

## Журнал (`JOURNAL.md`)
- В `.gitignore`. Новые записи сверху, формат `## YYYY-MM-DD: <тема>`. Тренировки, самочувствие, боль (0–10), корректировки.

## Дисклеймер
Не медицинское приложение, не заменяет врача/тренера. Прогрессия — постепенно; при боли прекращать.

# setup

Первичная настройка трекера тренировок (приложение на Apps Script + Google Sheets, `projects/training/app`).

## Триггеры
- «настрой трекер», «первичная настройка тренировок», `/setup`

## Алгоритм

### 1. Приветствие
Кратко объясни шаги: профиль → таблица/приложение → PRIVATE.md → первый план.

### 2. Профиль (одним блоком, отвечать как удобно)
- Имя, возраст, рост, вес.
- Уровень (новичок / возвращаюсь после перерыва / средний / продвинутый), частота (реком. 3 силовых + 2 кардио).
- Цели (здоровье/масса/сила/похудение/реабилитация).
- Ограничения и травмы (плечо, колени, поясница…) — какие движения запрещены/дискомфортны.
- Предпочтения (любимые/нелюбимые упражнения), оборудование, зал.

### 3. Таблица и приложение
- Спросить ссылку на Google-таблицу → извлечь `spreadsheet_id`.
- `list_sheets` — проверить доступ MCP.
- Убедиться, что есть вкладка **«Сессии»** с заголовками (см. `app/DATA-CONTRACT.md`); если нет — создать (`addSheet` + заголовки).
- Деплой приложения: указать на `projects/training/app/README.md` (clasp/ручной), получить URL веб-приложения.

### 4. Сгенерировать PRIVATE.md
На основе `PRIVATE.example.md`: профиль, `spreadsheet_id`, URL веб-приложения, `SHARED_SECRET` (опц.), ограничения. Персональное — только сюда (в `.gitignore`).

### 5. Создать первый план — **вызвать скил `new-plan`**
Передать в `new-plan` собранный профиль/ограничения и цель. Он задаст уточнения, подберёт упражнения, создаст вкладку `План: <название>` с заметками-контекстом.

### 6. Итог
Сводка: профиль, таблица подключена, приложение развёрнуто (URL), `PRIVATE.md` создан, план `План: <название>` готов.
Следующие шаги: открыть URL на телефоне → ⚙ выбрать план → тренироваться → завершить (поле «как прошла») → `/plan-next-week` для следующей недели.

## Правила
- Не давать медицинских назначений; при тревожных симптомах — к врачу.
- Итеративность: предлагать и ждать подтверждения.
- Персональные данные — только в `PRIVATE.md`.

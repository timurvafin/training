# plan-next-week

Прогрессия существующего плана: факт из «Сессии» → новая неделя в `План: <name>`.

## Триггеры
- «спланируй следующую неделю», «следующая неделя тренировок», «прогрессия»
- `/plan-next-week <план, каденс, самочувствие>`

## Формат данных (актуальный — `projects/training/app/DATA-CONTRACT.md`)
- **Планы = вкладки `План: <name>`.** Колонки: `A=День | B=Упражнение (ключ) | C=Разминка(да/пусто) | D+=Неделя N`. Строка = подход. Ячейка `повт×вес` (`8×120`, `10`, `8–12×40`, `—`=нет подхода). Дни — через пустые строки.
- **Факт = «Сессии»**: `ИД сессии | План | Неделя | Дата | День | Упражнение | № подхода | Разминка | Повторы | Вес | RPE | Заметка | Статус | Самочувствие | Время записи`.
- **Заметки-контекст** упражнений — комментарии (cell notes) на колонке B; обновляй их вместе с новой неделей.

## Шаги

### 1. Уточнить контекст
- **Какой план** (`План: …`)? Если не задан — спросить / последний используемый.
- **Каденс**: вся неделя или конкретный день?
- **Самочувствие**: колонка `Самочувствие` в «Сессии» (RPE всей тренировки 1–10) = RPE недели. Дополнительно уточнить боль/сон/настрой при необходимости.

### 2. Прочитать факт
- `get_sheet_data` «Сессии» → фильтр по `План` и последней `Неделя`/`Дате` (+ `День` при по-дневном). Сгруппировать по `Упражнение` → рабочие подходы (`Разминка` пусто): Повторы/Вес/RPE.

### 3. Прочитать план
- `get_sheet_data` «План: <name>». Недели — с колонки D. Последняя = крайняя правая с данными; новая = следующая справа.

### 4. Применить прогрессию
- **Двойная прогрессия:** все рабочие подходы на верху диапазона с запасом (RPE ≤ 8) → +вес (ноги +5–10, верх +2.5–5, hip thrust +5), повторы → низ. Иначе тот же вес, +1–2 повтора.
- **Самочувствие/RPE:** 1–4 агрессивнее; 5–6 стандарт; 7–8 или боль — консервативно; 9–10 — deload.
- **Deload каждые 4–6 недель:** −30–40% объёма, −20–30% веса.
- **Ограничения (`PRIVATE.md`):** жимы/подтягивания RIR 2–3; поясница; плечо. Разминки пересчитать от нового рабочего веса.

### 5. Показать сводку и подтвердить
`Упражнение | Последняя неделя | Новая неделя | Комментарий`. Спросить «Записать новую неделю?».

### 6. Записать новую колонку
- Заголовок новой колонки (первая свободная, D+) = `Неделя N+1`.
- `update_cells`/`batch_update` — для каждой строки-подхода ячейка `повт×вес` (разминки — пересчитанные). По-дневно: только строки нужного `Дня`. Прошлые колонки не трогать, пустые строки-разделители пропускать.
- **Обновить заметки-контекст** упражнений (cell notes, кол. B) при изменениях: новая логика прогрессии, разогрев от нового веса, технические напоминания.

### 7. После
- Приложение подхватит новую неделю (по умолчанию крайняя правая). Проверить через `getPlan`/`list_sheets`.
- Значимые решения — в `projects/training/JOURNAL.md`.

## Пример
`/plan-next-week fullbody, вся неделя` или `/plan-next-week fullbody, только День 1, чувствую себя хорошо`.

# new-plan

Создание новой программы тренировок для трекера (приложение на Apps Script + Google Sheets, `projects/training/app`).

## Триггеры
- «создай план тренировок», «новый план», «сделай восстановительный план»
- `/new-plan <описание>` · вызывается из `setup` для первого плана

## Формат данных (актуальный — `projects/training/app/DATA-CONTRACT.md`)
- **План = вкладка `План: <название>`.** Колонки: `A=День(номер) | B=Упражнение (ключ) | C=Разминка(да/пусто) | D+=Неделя N`.
- **Строка = один подход** (упражнения дублируются по подходам, вкл. разминочные). Ячейка = `повт×вес` (`8×120`, `10` без веса, `8–12×40`, `12×90lbs`, `10×тонк`; `—`/пусто = подхода нет).
- **Дни разделены двумя пустыми строками.** `spreadsheet_id` — в `PRIVATE.md`.
- **Заметки-контекст** — комментарии (cell notes) на колонке B первой строки упражнения: отдых, RIR, техника, разогрев, логика прогрессии. Приложение показывает их 💡 во время тренировки; агент читает при следующей прогрессии.

## Алгоритм

### 1. Уточнить параметры (AskUserQuestion, если не в аргументах)
- **Название плана** → вкладка `План: <название>` (напр. `fullbody`, `Восстановление`).
- **Цель:** восстановление после паузы / масса / сила / поддержка / реабилитация.
- **Дней в неделю** (обычно 3 full-body) и **сколько недель прописать вперёд** (1–4).
- **Ограничения/предпочтения** — из `PRIVATE.md`, при необходимости уточнить.

### 2. Проанализировать прошлое
- `get_sheet_data` «Сессии» → последние рабочие веса/повторы по упражнениям (отправная точка).
- Нет «Сессии» — посмотреть существующие планы `План: …` и/или legacy «Активную».
- **Возвращение после паузы** → стартовать ниже (−40–50%), выше повторы, RIR 3–4, 2 подхода, постепенный набор по неделям.

### 3. Подобрать упражнения
- Full-body, 4–6 упр./день. Базовые многосуставные в приоритете; баланс тяг/жимов ~2:1 (плечи). Каждая группа ≥2 раза/нед.
- Роли повторяющихся: День 1 тяжёлый, День 2 лёгкий/памп, День 3 средний.
- **Ограничения** (`PRIVATE.md`): плечо → наклонный жим/нейтральный хват, без горизонтальных жимов; поясница → РТ/hip thrust осторожно.
- Разминочные подходы — отдельными строками (`Разминка = да`): тяжёлые 40/65/85%, средние 50/75%.

### 4. Показать и согласовать
Таблица `День | Упражнение | Разминка | Неделя 1..N | Почему`. Итерировать до подтверждения.

### 5. Создать вкладку и записать
- `batch_update` → `addSheet` title `План: <название>` (если есть — спросить про перезапись).
- `update_cells`: строка 1 — заголовки (`День | Упражнение | Разминка | Неделя 1 | …`); далее строки-подходы по дням, **дни через 2 пустые строки**.
- **Заметки-контекст:** `batch_update` → `updateCells` с `fields: "note"` на колонке B (Упражнение), первой строке каждого упражнения — отдых, RIR, техника, разогрев, правило прогрессии. (rowIndex/colIndex 0-based; колонка B = startColumnIndex 1.)
- Не трогать «Активную» и другие планы.

### 6. Проверка
- `list_sheets` / приложение `listPlans` — план виден как `План: <название>`.
- Сообщить user: приложение → ⚙ Настройки → выбрать план → Неделя/День.

## Правила
- Не давать медицинских назначений; при тревожных симптомах — к врачу.
- Персональные данные — только в `PRIVATE.md`.
- Прогрессия консервативная и безопасная (RIR, без отказа на жимах/подтягиваниях).

# Контракт данных — HTML-трекер тренировок

Эталон форматов вкладок и JSON-обмена между агентом, Apps Script и приложением.
Источник правды по структуре. При изменении — править здесь и в `logic.ts`/`server.ts`/`ui/src/` синхронно.
Контракт обмена (`getPlan`/`saveSession`/`saveCardio`) при миграции UI на Svelte **не менялся**.

## Таблица

- **spreadsheet_id**, **URL веб-приложения**, **SHARED_SECRET** — в `PRIVATE.md` (не в репо).
- Вкладка «Активная» — **legacy** (исходная история), новым флоу не используется.
- «План» (широкая, недели по горизонтали, по подходам) и «Сессии» (факт по подходам).

## Планы = вкладки «План: <название>»

**Правило:** каждый план — отдельная вкладка с именем `План: <название>` (напр. `План: fullbody`).
`listPlans()` возвращает все такие вкладки; в приложении план выбирается в настройках ⚙ (смена плана — перечитывает данные). Прочие вкладки («Активная», «Сессии», черновики) под правило не попадают.

## Формат вкладки «План: …»

Похожа на «Активную»: недели по колонкам, но строки развёрнуты **по подходам** (вкл. разминочные).
Агент дописывает новую неделю новой колонкой справа; ячейка = `повт×вес`.

```
Колонки:  A=День(номер) | B=Упражнение (он же ключ) | C=Разминка(да/пусто) | [D=Отдых(сек, опц.)] | D+/E+=Неделя N (повт×вес)
Строка   = один подход. Упражнения дублируются по подходам; порядок строк = номер подхода.
Дни разделены двумя пустыми строками (косметика; парсер пустые строки пропускает).
```

> Заголовки структурных колонок (A–C) и язык — **косметика**: парсер читает их по позиции, а не по названию. Недели читаются по их меткам.
>
> **Колонка «Отдых» (D) — опциональная**, с авто-детектом (backward-compat). Парсер смотрит заголовок D (`header[3]`, trim+lowercase): если это `отдых`/`rest` (в т.ч. `отдых, с`, `rest, sec` — по префиксу) → колонка D = отдых, **недели сдвигаются на E+**. Иначе колонка отдыха отсутствует и недели читаются **с D+**, как раньше. Старые планы без «Отдых» парсятся без изменений.

- `День` (A) — номер дня (1/2/3). Группирует дни (вкладки в приложении: «День N»).
- `Упражнение` (B) — название; **оно же ключ** связи план ↔ «Сессии». При переименовании авто-связь истории рвётся (скил пере-сопоставит при перезаписи плана).
- `Разминка` (C) — `да` (разминочный подход) / пусто (рабочий). Разминки тоже трекаются.
- `Отдых` (D, опц.) — длительность отдыха после подхода в **секундах** (`120`, `120с`, `120 sec` — читается число). Пусто/`—` = нет значения (`set.rest` отсутствует). Колонка появляется только при заголовке `Отдых`/`Rest`.
- `Неделя N` (D+ без «Отдыха» / E+ с «Отдыхом») — ячейка `повт×вес` (`8×190`, `10` без веса, `12×90lbs`, `8–12×40`, `10×тонк`). `—`/пусто = подхода в эту неделю не было.

**Пример вкладки с «Отдых»:**
```
День | Упражнение      | Разминка | Отдых | Неделя 1 | Неделя 2
1    | Жим лёжа        | да       | 45    | 12×40    | 12×40
1    | Жим лёжа        |          | 120   | 8×60     | 8×62.5
1    | Жим лёжа        |          | 150   | 8×62.5   | 8×62.5
```
(Без колонки «Отдых» — те же строки, но столбец D опущен, недели начинаются с D.)

## Вкладка «Сессии»

Append-only. Одна строка = один подход (включая разминочные).

```
ИД сессии | План | Неделя | Дата | День | Упражнение | № подхода | Разминка | Повторы | Вес | RPE | Заметка | Статус | Самочувствие | Время записи
```

> Заголовки — косметика: строки пишутся по позиции (`buildSessionRows`). Порядок колонок фиксирован. **15 колонок.**
> `Самочувствие` — RPE всей тренировки (1–10), session-level, денормализовано на каждую строку.

- `ИД сессии` (session_id) — UUID, один на тренировку (`crypto.randomUUID()`). Ключ идемпотентности.
- `plan` — метка плана (из настроек приложения ⚙, по умолч. `fullbody`).
- `week` — выбранная неделя (метка из «План», напр. «Неделя 16 (план)»).
- `date` — `YYYY-MM-DD` день тренировки. `day` — `day_name` («День 1»).
- `name` — название упражнения (ключ связи с «План»).
- `set_index` — № записанного подхода внутри упражнения (1-based).
- `is_warmup` — `yes`/пусто. `reps`/`weight` — свободный текст (числа, диапазоны, единицы `90lbs`, бэнды `тонк`). `rpe` — 1–10/пусто. `status` — `completed`/`partial`. `saved_at` — ISO 8601 UTC.

## JSON-контракт приложение ↔ Apps Script

### `getPlan()` → объект (все недели сразу)

```json
{
  "weeks": ["Неделя 14", "Неделя 15 (разгрузка)", "Неделя 16 (план)"],
  "days": [
    {
      "day_id": "1", "day_name": "День 1",
      "exercises": [
        {
          "exercise_id": "Рычажная тяга (опора грудью)", "name": "Рычажная тяга (опора грудью)",
          "sets": [
            { "is_warmup": true,  "rest": 45,  "byWeek": { "Неделя 14": {"reps":"10","weight":"11"}, "Неделя 16 (план)": {"reps":"10","weight":"11"} } },
            { "is_warmup": false, "rest": 120, "byWeek": { "Неделя 14": {"reps":"8","weight":"30"},  "Неделя 16 (план)": {"reps":"10","weight":"30"} } }
          ]
        }
      ]
    }
  ]
}
```

> `set.rest` (число секунд) — **опциональное** поле подхода. Присутствует только если в плане есть колонка «Отдых» и ячейка заполнена; для старых планов (без колонки) или пустых ячеек — отсутствует. Приложение подставляет дефолт при отсутствии.

### `saveSession(payload)` ← объект

```json
{
  "shared_secret": "", "session_id": "uuid",
  "plan": "fullbody", "week": "Неделя 16 (план)", "date": "2026-06-04", "day": "День 1",
  "status": "completed",
  "sets": [
    { "name": "Рычажная тяга (опора грудью)", "set_index": 1,
      "is_warmup": true, "reps": "10", "weight": "11", "rpe": "", "note": "" }
  ]
}
```

### Ответ `saveSession`

- Успех `{ "ok": true, "written": N }` · Повтор `{ "ok": true, "written": 0, "duplicate": true }` · Отказ `{ "ok": false, "code": "...", "error": "..." }`
  (коды: `forbidden`, `bad_payload`, `no_session_id`, `too_long`, `no_sets`, `too_big`, `bad_set`, `bad_reps`, `bad_weight`, `bad_rpe`, `bad_status`, `bad_feel`, `busy`, `no_sheet`, `exception`)

## Кардио (отдельный лист «Кардио»)

Кардио — **одна запись с метриками**, а не подходы. Хранится отдельно от силовых «Сессий».

**День — кардио**, если его название (`day_id`) начинается с «Кардио» (напр. `Кардио 1`). В плане у такого дня одна строка-«упражнение» (`Кардио зона-2 …`), ячейка недели = целевая длительность (`40 мин`). Приложение для кардио-дня показывает **кардио-форму** вместо подходов: Вид (select) · Длительность (мин) · Ср. пульс (опц.) · RPE.

Лист **«Кардио»** (11 колонок):
```
ИД сессии | План | Неделя | Дата | День | Вид | Длительность (мин) | Ср. пульс | RPE | Заметка | Время записи
```

### `saveCardio(payload)` ← объект
```json
{ "shared_secret": "", "session_id": "uuid", "plan": "Восстановление", "week": "Неделя 1",
  "date": "2026-06-05", "day": "Кардио 1", "type": "Эллипс", "duration": "40", "hr": "128", "rpe": "5", "note": "" }
```
Ответ — как у `saveSession`. Коды: `forbidden`, `bad_payload`, `no_session_id`, `no_duration`, `bad_duration`, `bad_hr`, `bad_rpe`, `busy`, `no_sheet`, `exception`. Идемпотентность — по `session_id` в колонке A «Кардио».

## Поведение приложения

- **Шапка:** слева — крупный **селект НЕДЕЛИ** (`week-select`, на месте бывшего заголовка-программы; недели реактивно из текущего плана `app.plan.weeks`), справа — кнопки финиша (флаг) и настроек **⚙**. Отдельного заголовка-программы в шапке нет. Какой день открыт — определяет **активная вкладка**. Шапка **липкая** (селект недели + вкладки + статус-строка зафиксированы; скроллится только лента контента).
- **Выбор:** **Неделя** — в **шапке** (селект `week-select`, по умолчанию последняя). **План** — в настройках **⚙**; смена плана обновляет список недель в шапке. День — вкладки. Смена **плана** перечитывает данные с экраном загрузки (только онлайн); смена **недели** — мгновенно из кэша `getPlan` (все недели уже загружены, refetch нет; `flushSync` для немедленной перерисовки дней/подходов). Смена недели/дня/плана с несохранённым прогрессом — с подтверждением. Завершённый (залоченный) день не мешает переключиться на другой: новый день/неделя начинают **свежую сессию**.
- **Экран загрузки** — полноэкранный оверлей (бренд + прогресс) на первом старте и при смене плана.
- **Подходы:** разминки — метка 🔥 (из плана, `is_warmup`), тоже трекаются. **Тап по номеру (#) переключает тип подхода** (разминка ↔ рабочий, `toggleWarmup`) — таймер он не запускает. **RPE** — селект (6–10) или пусто. Число подходов — степпер **−/+**.
- **Таймер отдыха** — длительность на подход (колонка «Отдых», иначе дефолт). Авто-старт при ✓ **любого** подхода (рабочего или разминки — у разминок в плане тоже своя длительность отдыха). Док снизу: −15 / пауза / +15 / пропустить; превышение отдыха — счёт вверх (янтарный). Состояние по абсолютным моментам (`Date.now()`), переживает блокировку экрана и перезагрузку.
- **Завершение:** кнопка «Завершить» — **в шапке** (синий флаг рядом с ⚙). Пока отправлять нечего (силовая без единого ✓/повтора или кардио без длительности) — кнопка **disabled**. Отправка **один раз**; силовая — через оценку самочувствия (1–10). Во время отправки кнопка заблокирована (`_submitting`); после успеха UI текущей сессии блокируется (перезаписать нельзя — идемпотентность по `session_id`), кнопка «Начать заново» стартует новую сессию.
- **Статус синка** (строка под вкладками): в черновике (idle) **скрыта** — пока копишь подходы, плашки нет. Появляется только при отправке: «Отправляем…» → «Отправлено», либо «Оффлайн — отправится позже» с **кнопкой ретрая ↻ внутри самой строки**.
- **Offline** — состояние в `localStorage`; при отсутствии связи sync откладывается (↻ в статус-строке + авто-ретрай по `online` и при перезапуске).

## Идемпотентность и безопасность

- `saveSession` перед записью проверяет `session_id` в колонке A «Сессии» → повтор не дублирует.
- Доступ web app «Только я» (Workspace блокирует анонимный); опц. `SHARED_SECRET` (Script Properties + ⚙). Текстовые ячейки экранируются от формул-инъекции.

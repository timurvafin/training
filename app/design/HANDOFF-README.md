# Handoff: Трекер силовых тренировок (мобильный веб-апп, тёмная тема)

## Overview
Мобильный веб-апп для записи силовых тренировок прямо в зале между подходами:
быстрый ввод подхода в 1–2 тапа, крупные тач-цели, тёмная тема, русский интерфейс.
Тренер-агент заранее пишет план в Google-таблицу; приложение показывает план на
сегодня и пишет факт обратно. Один экран = одна тренировка.

Есть две версии прототипа:
- **v1** — базовая (без таймера).
- **v2** — то же + **таймер отдыха с длительностью на каждом подходе** (рекомендуется как актуальная).

## About the Design Files
Файлы в этом пакете — **дизайн-референсы, сделанные в HTML/CSS/JS (React + Babel inline)**.
Это прототипы, показывающие желаемый вид и поведение, **а не продакшн-код для копирования**.
Задача — **воспроизвести эти экраны в существующем окружении вашего проекта** (React/Vue/
SwiftUI/native и т.д.) по принятым в кодовой базе паттернам и библиотекам. Если окружения
ещё нет — выбрать подходящий фреймворк и реализовать дизайн в нём.

Состояние и «загрузка из Google Sheets» в прототипе сымитированы (setTimeout). В реальном
приложении это: чтение плана из таблицы и запись факта обратно через бекенд/Apps Script.

## Целевой стек: Svelte + Vite (`@sveltejs/vite-plugin-svelte`, `vite-plugin-singlefile`)
Реализовать как Svelte-компоненты; финальная сборка — один self-contained HTML (singlefile),
что идеально ложится на «один экран = одна тренировка».

**Маппинг прототипа на Svelte:**
- **Токены** из `hifi-v2/theme.css` + компонентные стили из `hifi-v2/app.css` — перенести как есть
  в глобальный CSS (напр. `src/app.css`, импорт в корне). CSS-переменные на `:root` оставить —
  тема/акцент/тач-цели переключаются простой сменой переменных (см. ниже). Тёмная база — по умолчанию.
- **React-компоненты → Svelte:** `Header`, `DayTabs`, `SyncLine`, `ExerciseCard`, `SetRow`,
  `CardioForm`, `RestTimer`, `Footer`, `LoadingScreen`, `FinishSheet`, `SettingsSheet`,
  `EmptyDay` → одноимённые `.svelte` файлы. JSX-разметка почти 1:1 в Svelte-шаблон
  (`className`→`class`, тернарники → `{#if}` / `{:else}`, списки → `{#each}`).
  Поля Повт/Вес/Длительность/Пульс — нативные `<input>` с `inputmode` (не кастомная клавиатура).
- **Состояние:** прототипный `useState`/`useTweaks` → Svelte stores
  (`writable`) в `src/lib/stores.js`: `plan`, `activeId`, `week`, `status`, `sync`,
  `modal`, `loading`, `timer`. План-сид — из `hifi-v2/data.jsx` (поле `rest` на каждом подходе).
- **Таймер отдыха:** интервал на `setInterval` в `onMount`/эффекте store; при достижении 0 —
  `finished=true` и счёт `over` вверх (НЕ скрывать). Очистка интервала в `onDestroy`.
- **Поля ввода / модалки:** числовые поля — нативные `<input inputmode>`; модалки (завершение,
  настройки) — компоненты с `{#if}` + затемнение, анимации входа через Svelte `transition:fly`/`fade`
  (безопаснее inline-CSS-анимаций; entrance не «застрянет»).
- **Скейл телефона** под вьюпорт: `transform: translate(-50%,-50%) scale(s)` в `onMount`+`resize`
  (как в `hifi-v2/app.jsx`, эффект `fit`). Либо, если апп идёт на весь экран реального телефона —
  отказаться от рамки-устройства и сделать корневой контейнер `100dvh`.
- **Tweaks-панель НЕ переносить** — это инструмент прототипа. Но логику она задаёт полезную:
  смена акцента = установка `--accent`/`--on-accent`; AMOLED = класс на корне; тач-цели = `--tt`;
  автостарт таймера и оффлайн-режим — обычные настройки приложения.
- **Шрифты:** Hanken Grotesk + Space Grotesk. Для singlefile-сборки лучше заинлайнить шрифты
  (woff2 base64 или `@fontsource/...`), чтобы не тянуть Google Fonts из офлайна в зале.
- **Иконки:** инлайн-SVG из `hifi-v2/icons.jsx` → перенести в Svelte (напр. `<Icon name=.. />`)
  или заменить на вашу icon-библиотеку.

Прод-заметки: numeric-поля в реальном вводе — `inputmode="decimal"`; «Начать заново» в
завершённой записи стоит подтверждать; учесть `env(safe-area-inset-*)` для футера/шитов.

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы, интеракции. Воспроизводить
пиксель-в-пиксель средствами вашей дизайн-системы. Размеры даны под viewport 390×844 (iPhone).

### 5. Ввод чисел — нативная клавиатура телефона
Поля Повт/Вес/Длительность/Пульс — это обычные `<input>` (НЕ кастомный боттомшит).
Тап по полю фокусирует input и вызывает системную числовую клавиатуру.
- **Повт:** `<input inputmode="numeric" pattern="[0-9]*">` (целые). Очистка → подход не отмечен.
- **Вес:** `<input inputmode="decimal">`, поддерживает дробные (`62.5`); запятая нормализуется в точку.
  Единица `кг` — отдельный `<span>` справа; пустое значение для веса тела показывает плейсхолдер «своё».
- **Длительность/Ср. пульс** (кардио) — тоже `inputmode="numeric"` (пульс опционален, плейсхолдер «—»).
- Поле в покое: число `--text-3` (плановое, faint); при фокусе — рамка/тень `--accent`
  (`:focus-within`), `box-shadow: 0 0 0 4px --accent-tint`; после ввода значение становится ярче,
  подход помечается `touched`. Размер числа в поле подхода 17px, в кардио 22px (tabular-nums).
- В вебе важно: `inputmode` для правильной раскладки; для веса допускать `.`/`,`. Степперы/быстрые
  чипы не используются — ставка на скорость нативного ввода и предзаполненный план.

## Rest Timer (только v2) — длительность на каждом подходе

**Данные:** у каждого подхода своё поле `rest` (секунды). Пример (Жим лёжа): разминка 45,
рабочие 120/120, последний 150. Изоляция ~60. Под номером подхода эта длительность видна (`120с`).

**Запуск:**
- Авто: при отметке ✓ **рабочего** подхода стартует отсчёт от `rest` ЭТОГО подхода
  (Tweak «Автостарт» вкл/выкл).
- Вручную: тап по ячейке номера подхода → старт его `rest`.

**Док-бар** над футером (`--surface`, border `--hair-2`, radius 18, тень):
- Верх: подпись `ОТДЫХ · <упражнение>` (12 uppercase, ellipsis) + «Пропустить».
- Центр: `−15` | крупный отсчёт `M:SS` (тап = пауза/продолжить, иконка ⏸/▶) | `+15`
  (кнопки adj 58×46; время — height 54, `--surface-2`).
- Прогресс-полоса (`--accent`), width = remaining/total.

**Переотдых (после нуля) — НЕ скрывать:**
- Бар переходит в **янтарный** (`--warn-tint`, border warn), короткий пульс тени (2 цикла).
- Подпись «ОТДЫХ ОКОНЧЕН · <упражнение>», справа «Готово».
- Центр: пауза | блок `+M:SS` (янтарный, счёт **вверх**) с подписью `сверх M:SS` | иконка skip.
- Полоса полная, янтарная.
- Логика: счётчик идёт вниз до 0 → `finished=true`, затем `over` инкрементируется вверх,
  пока пользователь не нажмёт «Готово»/skip. Пауза работает и в переотдыхе.

## Interactions & Behavior
- Тап по полю Повт/Вес → фокус нативного `<input>` + системная клавиатура; ввод сразу пишется в state и помечает подход.
- Выбор RPE → нативный `<select>` (пусто/6–10). Тап по ✓ → выполнено/нет (+ автостарт таймера в v2).
- Степпер ± добавляет/убирает подход.
- Переключение дней — мгновенное; статус сбрасывается в active.
- «Завершить тренировку» → модалка → «Отправить» → заблокированное состояние + баннер.
- «Начать заново» → сброс подходов, active.
- Шестерёнка → настройки; смена плана/недели → экран загрузки.
- Синхронизация НЕ на каждое изменение: во время ввода статус = `idle` (черновик). Единственная
  отправка факта — по «Завершить» (`sync` → `ok`). Оффлайн (Tweak): статус `off`,
  при завершении факт ставится в очередь; «Повторить»/↻ → `sync` → `ok`.
- Масштаб: телефон 390×844 центрируется и скейлится под вьюпорт через
  `transform: translate(-50%,-50%) scale(s)` (НЕ обрезается).

**Анимации:** избегать entrance-анимаций, прячущих контент в `from` (opacity:0 / translateY 100%) —
они замораживаются на первом кадре в неактивном фоне. Спиннер/пульс точки — ок (видимы в любом кадре).

## State Management
Ключевое состояние (React в прототипе):
- `plan` (дни → упражнения → подходы `{reps, weight, rpe, warm, rest, done, touched}`), `activeId`, `week`.
- `status`: `active | completed`. `sync`: `idle | sync | ok | off` (по умолчанию `idle`; меняется только при завершении).
- `modal`: `finish | settings`. `rating`, `reveal`, `planIdx`. (Поля ввода — нативные `<input>`,
  отдельного состояния клавиатуры нет; вес во время набора держит локальный буфер строки для дробных.)
- `loading: {on, label, pct}` (последовательность шагов).
- `timer: {id, exName, total, remaining, running, finished, over}` (только v2).
Данные: в реальном приложении план тянуть из Google Sheets, факт писать обратно (секрет записи).
